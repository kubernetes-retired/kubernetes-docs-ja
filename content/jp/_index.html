Kubernetesは、コンテナ化したアプリケーションのデプロイ、スケール、管理を自動化するオープンソースのシステムです。

Kubernetes はコンテナをグループ化し、管理と発見が簡単になるよう、アプリケーションを論理的な集まりに構成します。 Kubernetes の基礎を作り上げるのは、 Google において 15 年にわたるプロダクション・ワークロードを動かしている経験 と、コミュニティからのアイディアおよび経験の最善の組み合わせが一体となったものです。

世界規模のスケール

oogle が毎週 10 億ものコンテナを動かすのと同じ方式で設計しているため、Kubernetes は皆さんの運用チームを増員しなくてもスケールできます。

成長は継続

ローカルにおけるテストやグローバルなエンタープライズにおいても、Kubernetes の複雑さが増す心配は不要であり、アプリケーションを一貫して簡単に供給し、皆さんと共に柔軟に成長します。

あらゆる場所で動作

Kubernetes はオープンソースであり、オンプレミス、ハイブリッド、あるいはパブリッククラウド基盤の利点を自由に得られるため、ワークロードを効率的かつ心配する必要なくどこでも移動可能にします。

Kubernetes 機能

自動的に集約

リソース要件と他の制限に基づき、可用性を犠牲にせず、コンテナを自動的に配置します。リソースの稼働率を高め、リソースを更に制限するために、クリティカルおよびベスト・エフォートなワーククロードを組み合わせます。

自己修復 （Self-healing）

障害が起こればコンテナを再起動し、ノードが壊れればコンテナの再配置と再スケジュールをし、定義したヘルスチェックの応答がなければコンテナを停止し、クライアントに提供する準備が整わなければ広報(advertise)しません。

水平スケール

簡単なコマンドか UI あるいは CPU 使用率に基づき、アプリケーションを自動的にスケールアップ・ダウンします。

サービス・ディスカバリと負荷分散

サービス・ディスカバリ機構になれていなくても、アプリケーションに手を加えることなく利用できる。Kubernetes はコンテナに対して IP アドレスとコンテナ群に対する１つの DNS 名をあたえ、相互に負荷分散できるようにします。

自動的なロールアウトとロールバック

Kubernetes はアプリケーションの正常性を確保したまま、全てのインスタンスを停止せず、アプリケーションや設定に対する変更を徐々に適用する。もし何か問題が起これば、Kubernetes は変更に対してロールバックすします

シークレット と 設定 の管理

シークレットとアプリケーション設定の展開と更新には、イメージの再構築やスタック設定内でシークレットを露出しない。

ストレージ・オーケストレーション

ローカル・ストレージや GCP や AWSどのパブリック・クラウド・プロバイダ、あるいは NFS や iSCSI、Gluster、Ceph、Cinder、Flocker のような、選択に応じたストレージ・システムを自動的にマウントします。

バッチ実行

サービスだけでなく、Kubernetes はバッチや CI ワークロードなども管理でき、必要があれば障害時にコンテナを置き換えられます。
